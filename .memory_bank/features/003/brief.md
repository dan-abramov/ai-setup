Какую проблему решаем:
В 100% сессий после закрытия вкладки пользователь не может повторно открыть ранее сгенерированное описание задачи.

Для кого:
Для программистов Ruby уровня junior/middle, которые проходят первое задание в MVP.

Откуда взялась задача:
Это задача из плана подготовки MVP. По текущей аналитике completion rate первого задания уже снизился на 3 п.п., что создает риск невыполнения KPI MVP к 1 мая 2026 года.

Что мы хотим получить:
Не менее чем в 95% сессий пользователь может повторно открыть ранее сгенерированное описание задачи после закрытия вкладки.
